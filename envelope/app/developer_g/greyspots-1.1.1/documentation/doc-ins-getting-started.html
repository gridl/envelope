Getting Started

Intro:

To get started on your first application with Envelope you need to be familier with the workflow of Envelope. The workflow can be broken down into three parts:

  1. Postgres Files
  2. HTML Files
  3. Adding A Link To The Main Menu
  4. Releasing Your App

Each step will be covered in the sections below. Not listed in these steps is Familiarization and Planning. Please look at all of the elements at your disposal so that you can make an actionable plan and then begin the development of your application.

Postgres Files:

The first thing you need in a application is your backend. To start on the backend follow these steps:

  1. Go to the main menu
  2. Click the "SQL Manager" button on the left side
  3. Click the "New File" button at the top of the left panel
  4. Write a name for your file
  5. Choose "Single use script" in the "File Type" select box
  6. Click "Create"

Once you have finished these steps you can begin creating a Postgres schema for your application. Your schema should have a dedicated postgres group that allows users to use the schema.

To test your sql: click the play button in the header on the right side of the screen or click the "Pull" button. Clicking the play button will run your SQL against the developement database, clicking the "Pull" button will get a fresh copy of the production database and run the sql files on that. When you release your application the files you create in the SQL Manager will be run against the production database. After release all "Single use script"s will be deleted.

HTML Files:

Once you have started your backend you can start your front end. To start on the frontend follow these steps:

  1. Go to the main menu
  2. Click the "File Manager" button on the left side
  3. Click the "New Folder" button in the header
  4. Name your file after the group that has access to your application's schema
  5. Click the new folder in the list
  6. Click the "New File" button in the header
  7. Name your file after the purpose of the screen or call it "index.html"
  8. Click the pencil icon next to the new file in the list
  9. In the code start typing in "document" and choose the "Document Start" snippet

You now have an HTML file to build your screen in. Your elements will go in the "<body>" tag. Your JavaScript will go in a "<script>" tag within the "<head>" element. Your CSS will go in a "<style>" tag within the "<head>" element.

Adding A Link To The Main Menu:

Once you have your HTML file working you can add a link for it to the main menu. To add a link to the main menu follow these steps:

  1. Open the "File Manager"
  2. Go to the "all" folder
  3. Click the pencil icon next to the "index.html" file
  4. Find the text '<div id="user-app-list-container">'
  5. Inside the "user-app-list-container" div element create a new <gs-button> element
  6. Fill in the text of the button
  7. Add the "target" attribute and set it to "_blank"
  8. Add an "href" attribute that links to your app's main page
  9. If you want to resrict the button to a certain group or user:
        To restrict to a group: set the "data-group" attribute to what group you want to see it
        To restrict to a user: set the "data-user" attribute to what user you want to see it

Releasing Your App:

Once you have gotten your application to a minimal working state you can release it. To release your application follow these steps:

  1. Go to the main menu
  2. Click the "Pull" button
  3. Click the "Release" button
  4. Name the release
  5. Click "Ok"
Once you have released: production users with the appropriate permissions can now use your application.
